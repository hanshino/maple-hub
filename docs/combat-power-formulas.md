## 一、角色屬性 (Stats) 的分層與計算

角色的最終屬性（STR、DEX、INT、LUK）分為四個運算層次：

| 層次                   | 說明                          | 範例來源                               |
| :--------------------- | :---------------------------- | :------------------------------------- |
| 裸屬性 (Pure Stats)    | 升級時 AP 分配的基礎屬性      | AP 點數                                |
| 附加屬性 (Added Stats) | 非百分比、但能受 % 加成的屬性 | 裝備固定數值、楓之谷勇士技能           |
| 百分比屬性 (Stats%)    | % 數加成                      | 潛能、Link 技能                        |
| 最終屬性 (Final Stats) | 不受 % 加成的固定數值         | ARC、AUT、聯盟佔領、超級屬性、內在能力 |

> 後期 ARC/AUT 越高，裝備上 % 屬性的相對收益會隨之遞減。

### 總屬性公式

```
總屬性 = floor((裸屬性 + floor(附加屬性)) × (1 + 百分比屬性 / 100)) + 最終屬性
```

> **參考**: Wiki 傷害公式文檔；Calculator `Charactor.py` -> `getAPinfo` 方法
> Calculator 中簡化為 `AP = AP_Base × (1 + AP%) + AP_Flat`，未區分「最終屬性」層。

## 二、攻擊力計算

魔法師計算魔法攻擊力，其餘職業計算物理攻擊力。攻擊力同樣分為三層：

- **附加攻擊**: 裝備、技能提供的固定攻擊力
- **百分比攻擊**: 潛能等提供的 % 攻擊加成
- **最終攻擊**: 不受 % 加成的固定攻擊（如內在能力「每隔一定等級攻擊力增加」）

### 總攻擊力公式

```
總攻擊力 = floor(floor(附加攻擊) × (1 + 百分比攻擊 / 100)) + 最終攻擊
```

## 三、面板屬性攻擊力 (Damage Range)

```
面板攻擊力 = (主屬性 × 4 + 副屬性 × 1) × 武器係數 × 總攻擊力 / 100
```

> **參考**: Calculator `Charactor.py` -> `calcAttributeByClass` 方法

### 熟練度 (Mastery)

面板上限與下限的比例由熟練度決定，實際攻擊力在最大值與最小值之間隨機浮動。

## 六、傷害計算的增傷乘區

最終傷害數字由多個獨立乘區相乘決定：

### 6.1 總傷害 / Boss 傷害（加法區）

傷害%、Boss 傷害%、對一般怪物傷害% 三者**加法合併**後，作為一個乘區。

```
傷害乘區 = 1 + (傷害% + Boss傷% + 對一般怪物傷%) / 100
```

### 6.2 最終傷害（乘法區）

所有終傷來源互相**乘法疊加**，是最強的獨立乘區。

```
終傷乘區 = (1 + 終傷A%) × (1 + 終傷B%) × ...
```

### 6.3 爆擊傷害

```
爆傷乘區 = 1 + 爆擊傷害%
```

> Calculator 中使用基礎爆傷 1.35（即 35%），實際遊戲中爆傷為 20%~50% 浮動，平均約 35%。

### 6.4 無視防禦

各來源的無視防禦為**乘法疊加**：

```
總無視 = 1 - (1 - 無視A) × (1 - 無視B) × ...
```

實際對怪物的防禦削減：

```
實際怪物防禦 = 怪物防禦% × (1 - 總無視%)
```

> **參考**: Calculator `Charactor.py` -> `calcIgnore` 方法

### 6.5 總增幅比例（效益計算用）

計算換裝效益時，將各乘區的提升比率相乘：

```
總增幅 = (屬性_new / 屬性_old)
       × (攻擊_new / 攻擊_old)
       × ((1 + 攻擊%_new) / (1 + 攻擊%_old))
       × ((1 + 傷害_new + Boss_new) / (1 + 傷害_old + Boss_old))
       × ((1.35 + 爆傷_new) / (1.35 + 爆傷_old))
       × ((1 - 怪防 × (1 - 無視_new)) / (1 - 怪防 × (1 - 無視_old)))
       × ((1 + 終傷_new) / (1 + 終傷_old))
```

> **參考**: Calculator `Charactor.py` -> `calcImprove` 方法

## 八、環境與特殊機制

### 8.1 屬性抗性

多數中後期 Boss 具備全屬性抗性，使最終傷害**強制減半**。

### 8.2 地圖壓制力

| 條件                        | 效果               |
| :-------------------------- | :----------------- |
| 星之力/ARC/AUT 低於地圖要求 | 傷害受嚴重懲罰     |
| ARC 達到要求的 1.5 倍       | 最高 150% 傷害增幅 |

### 8.3 多段攻擊快照

施放多段技能時，系統一次性鎖定當下面板狀態，但每段傷害數字與是否爆擊各自獨立隨機。

### 8.4 技能倍率提升

- `+%p` 標註：與原技能倍率**直接相加**（250% + 100%p = 350%）
- 未標註 `%p`：通常視為總傷害%，與裝備上的總傷/B傷**加法稀釋**

## 九、特殊技能類型的傷害判定

| 技能類型              | 判定差異                                                                 |
| :-------------------- | :----------------------------------------------------------------------- |
| 主動技能 / 一般召喚物 | 正常套用所有公式                                                         |
| 持續傷害 (DoT)        | 以最大面板計算（不受熟練度浮動）；不吃總傷%、B傷%、終傷%、爆傷、無視防禦 |
| 靈魂武器召喚物        | 對 Boss 傷害強制減半；不吃 B傷%、無視防禦、爆傷、屬性抗性                |

## 十、特殊職業機制

### 惡魔復仇者 (Demon Avenger)

- 裸血量（AP 分配的 HP）增加面板效益比其他來源**高 25%**
- 裝備與套裝的附加血量（非 % 數）效果**強制折半**

### 陰陽師 (Kanna)

- 裝備上的附加 MP（非 % 數）自動視為附加 HP 計算
- 新手技能「五行陰陽師」提供的攻擊力屬於「最終攻擊力」

## 十一、公式驗算（實測數據）

以角色「影之愛衣」（暗夜行者 Lv.295）的 API 數據進行驗算。

### 角色數值

| 項目       | 數值         |
| :--------- | :----------- |
| 主屬 LUK   | 92,434       |
| 副屬 DEX   | 9,423        |
| 攻擊力     | 13,903       |
| 武器係數   | 1.75（短劍） |
| 熟練度     | 85%          |
| 傷害%      | 95.00%       |
| Boss傷害%  | 531.00%      |
| 最終傷害%  | 94.81%       |
| 爆擊傷害%  | 123.25%      |
| 無視防禦率 | 96.20%       |

### 11.1 面板屬性攻擊力

API 回傳的「最高屬性攻擊力」包含傷害%和最終傷害%兩個乘區：

```
最高屬性攻擊力 = (主屬 × 4 + 副屬) × 武器係數 × 攻擊力 / 100 × (1 + 傷害%) × (1 + 終傷%)
```

驗算：

```
= (92434 × 4 + 9423) × 1.75 × 13903 / 100 × 1.95 × 1.9481
= 350,440,102

API 回傳值: 350,440,127（誤差 25，為取整累積）
```

最低屬性攻擊力 = 最高 × 熟練度%：

```
= 350,440,102 × 0.85 = 297,874,087

API 回傳值: 297,874,110（吻合）
```

### 11.2 戰鬥力

戰鬥力使用標準化武器係數 1.3，且乘區為 Boss傷害（非終傷）和爆擊傷害：

```
戰鬥力 ≈ (主屬 × 4 + 副屬) × 1.3 × 攻擊力 / 100 × (1 + 傷害% + Boss傷%) × (1 + 爆傷%)
```

驗算：

```
= (92434 × 4 + 9423) × 1.3 × 13903 / 100 × (1 + 0.95 + 5.31) × (1 + 1.2325)
= 1,110,711,664

API 回傳值: 1,089,125,287（比率 1.0198，約 2% 誤差）
```

> 2% 誤差可能來自：戰鬥力排除部分終傷來源、無視防禦率的折算、或取整差異累積。

### 11.3 驗算結論

| 公式               | 驗算結果                   |
| :----------------- | :------------------------- |
| 面板最高屬性攻擊力 | 完美吻合（誤差 < 0.0001%） |
| 面板最低屬性攻擊力 | 完美吻合（最高 × 熟練度%） |
| 戰鬥力             | 近似吻合（約 2% 誤差）     |

關鍵發現：

- **面板屬性攻擊力**包含「傷害%」和「最終傷害%」，但不包含 Boss傷害% 和爆擊傷害%
- **戰鬥力**使用標準化武器係數 1.3，包含「傷害% + Boss傷害%」和「爆擊傷害%」，但不包含終傷%
- 兩者的乘區組成不同，這解釋了為何面板高不一定戰鬥力高

## 十二、Preset 切換戰鬥力推算（實測驗證）

### 12.1 裝備系統結構

Nexon API 回傳的裝備資料結構：

| 欄位                          | 說明                                   |
| :---------------------------- | :------------------------------------- |
| `item_equipment`              | 當前實際穿戴的所有裝備（含獨立裝備）   |
| `item_equipment_preset_1/2/3` | 三套 Preset 各自的裝備（不含獨立裝備） |
| `preset_no`                   | 當前使用的 Preset 編號                 |

裝備分為兩類：

- **Preset 裝備**（24 件）：隨 Preset 切換而改變。Preset 1 為基礎，Preset 2/3 可覆蓋部分裝備
- **獨立裝備**（不隨 Preset 切換）：
  - **圖騰**（馬鞍、馴服的怪物）：不在任何 Preset 的 slot 中
  - **寶玉**（如伊妮絲的寶玉）：每個角色一個，獨立於 Preset 系統。API 的 slot 標記為「墜飾」，但它與墜飾1/墜飾2 是分開的，三者同時存在

判斷獨立裝備的規則：

1. `item_equipment_slot` 不存在於任何 Preset → 圖騰
2. 同一 slot 在 `item_equipment` 中出現多次 → 額外的是寶玉

### 12.2 反推計算法

已知 API 回傳的當前角色 stat 為真值，可反推出「非裝備常數」，再替換 Preset 裝備重算。

**公式模型：**

```
總屬性 = floor((C_base + 裝備固定值) × (1 + 裝備百分比%)) + C_final
```

- `C_base`：非裝備的受%加成基數（AP + 套裝全屬 + 聯盟佔領 + 神器 + Buff/技能等）
- `C_final`：不受%加成的最終屬性（符文 ARC/AUT）
- `裝備固定值`：`item_total_option` 的數值 + 潛能固定值
- `裝備百分比`：潛能的 % 屬性 + % 全屬性

**反推步驟：**

1. 計算符文最終屬性：`C_final = Σ(100 + symbol_level × 100)`（每顆符文）
2. 分析當前裝備的固定值和百分比總和
3. 反推：`C_base = (API_stat - C_final) / (1 + 裝備%) - 裝備固定值`
4. 切換 Preset 時替換裝備部分，保留獨立裝備和常數不變
5. 重算：`新stat = floor((C_base + 新裝備固定值) × (1 + 新裝備%)) + C_final`

### 12.3 API 資料來源

完整推算需要以下 API：

| API                           | 用途                           |
| :---------------------------- | :----------------------------- |
| `/character/stat`             | 當前角色屬性（真值）           |
| `/character/item-equipment`   | 裝備資料（含 Preset 1/2/3）    |
| `/character/symbol-equipment` | 符文等級（算 C_final）         |
| `/character/set-effect`       | 套裝效果（算 C_base 的一部分） |
| `/user/union-raider`          | 聯盟佔領加成                   |
| `/user/union-artifact`        | 聯盟神器加成                   |

### 12.5 注意事項

- 角色 Buff（如楓之谷勇士）不在任何 API 中，但被歸入 `C_base` 反推值，不影響 Preset 間的差異計算
- 傷害%、Boss傷%、終傷%、爆傷% 若也因 Preset 裝備潛能而改變，需一併重算（目前假設不變）
- 戰鬥力公式仍有約 2% 系統性誤差，面板公式則完全精確
- 符文屬性公式 `100 + level × 100` 為推算值，尚未從官方文檔確認
